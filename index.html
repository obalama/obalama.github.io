MTOP是什么？

无线端不能直接调用http接口，集团内部是走MTOP的接口。它替我们完成了权限校验，参数绑定等工作。

怎么接入？

  • 最全的MTOP文档(MTOP用户文档)
  • http服务接入MTOP(官方CF空间)
  • 服务提供方需配置MTOP(虽然不合理，但是目前是这样的)
  • hsf服务绑定
    • hsf返回值绑定
    • hsf参数绑定
      • map绑定的话虽然没有业务含义，但是可以的，传入map的json格式
      • POJO绑定的话将字段和api的参数绑定
      • MtopContext是一个ThreadLocal的变量，虽然可以再hsf服务中直接调用，但是最好还是不要使用，最好的办法是在MTop的配置页面中绑定。这个可以参考我的mtop.taobao.reserveOperateFacade.reserve接口。

后记

之后需了解一下http服务怎么接入。

缘起

由于在阿里没有自己从零搭建过一个全新的webx的系统，这次有机会自己搭了一个。遇到了诸多坑。记录一下。

deploy的时候的坑

deploy工程里面主要有两个文件，一个是pom.xml，

一个是assembly.xml,

其中，pom.xml里面finalName必须和应用名一样（这个必须），同时最好保持和assembly.xml里面的baseDirectory里面保持一致。否则会提示，“ERROR: /home/admin/mppweb/target/mppweb.tgz NOT EXISTS”已我的mppweb应用为例，pom.xml里面finalName必须为mppweb,assembly.xml里面baseDirectory为mppweb.war(有war后缀)。同时，assembly.xml里面directory必须保证和web工程打包生成的名字一样，即与web工程里面pom.xml里面配置的finalName一样。这里需要提到一点，那就是工程里面pom.xml中finalName表明了最终生成的包名（不带后缀，后缀由打包类型决定），如果不指定这个finalName,则打包的时候会带上版本信息（这个当时是一个大坑，因为这样的话deploy的时候directory必须跟上后缀，这显然是不好的）。
另外一个大坑值得一提，那就是他娘的，在打包的时候一定要设置自动解压，即在web工程的pom.xml配置插件的时候要如下设置（exploding设为true）,否则打包的时候就会失败提示“you must set at least one file maven assembly”，因为找不到可以打包的对象。
